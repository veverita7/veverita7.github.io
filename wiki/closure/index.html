클로저(Closure)

이 문서는 MDN의 클로저 문서를 정리한 내용입니다.

MDN에 보면 클로저는 함수와 함수가 선언된 어휘적 환경(the lexical environment)의 조합이다. 라고 정의하고 있다.

설명만 봐서는 무슨 의미인지 이해하기 어렵다.

클로저를 이해하기 위해서는 우선 어휘적 범위 지정(Lexical scope) 에 대해서 알아야한다.

어휘적 범위 지정 (Lexical scope)

렉시컬 스코프란 스코프가 함수/변수를 선언한 위치에 따라 스코프가 결정된다는 것을 말한다.

렉시컬 스코프와 관련된 간단한 예제를 보자.

  • inner 함수는 outer 함수 내부에서 선언되고 호출된다.

  • inner 함수는 자신이 속한 렉시컬 스코프인 outer를 참조할 수 있다.

  • 따라서 outer 함수는 outer 함수 내부의 count 변수에 접근할 수 있다.

렉시컬 스코프와 클로저

그럼 이제 클로저에 대해서 알아보자.

결론부터 말하면 위 예제에서 inner 함수가 클로저이다.

Java나 C 개발자가 보기에는 위의 결과가 이상하다고 생각할 수 있는데
왜냐하면 outer 함수를 실행하여 inner를 리턴받았음에도 불구하고
count에 접근이 가능하고 count의 값이 유지되기 때문이다.

그 이유는 inner 함수가 outer 함수의 count 변수를 참조하는 상황에서
외부의 변수 func에 inner 함수가 할당되면서 해당 Context가 유지되기 때문이다.

클로저를 정리하면

  1. 내부 함수가 유효한 상태일 때 == var func = outer();

  2. 외부 함수가 종료되어도 == return inner;

  3. 외부 함수의 객체가 내부 함수에 참조되면 == console.log(++count);

  4. 내부 함수가 스코프 체인을 통해 외부 함수의 객체를 참조할 수 있다.
