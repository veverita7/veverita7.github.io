module

  • 독립 가능한 기능 단위
  • 표준 ES2015 모듈 지원
  • 장점
    • 유지보수 용이
    • 전역 스코프 오염 방지
    • 재사용성 향상
  • 모듈러 프로그래밍 진행과정
    1. 모듈 식별
    2. 모듈 분리 선언
    3. 모듈 외부 공개
  • 내부모듈과 외부모듈
    • 내부모듈: 네임스페이스 기준
    • 외부모듈: 모듈 파일 기준

internal module

  • 네임스페이스 기준
    • 같은 네임스페이스에서는 클래스나 함수, 변수 중복 불가
    • 네임스페이스는 전역 스코프지만 독립된 공간

external module

  • export 키워드로 선언
  • 외부 모듈의 네임스페이스는 파일 내부로 제한
  • module 옵션으로 컴파일
    • tsc –module <형식>
    • commonjs, amd, system, umd, es2015/es6

namespace

multi namespaces in a file

  • 컴파일 후 Park과 Jung이 var 선언자로 선언되어 호이스팅
  • 순서와 상관없이 호출이 가능

one namespace in multi files

  • 참조 경로는 트리플 슬래시로 선언
  • car2.ts만 컴파일하면 참조하는 car1.ts도 컴파일됨
  • 여러 파일로 나누는 경우 그냥 컴파일하면 작동 불가
    • tsc –out out.js car2.ts
    • 파일 하나로 묶어야함

namespace module

  • 네임스페이스를 export로 모듈로 선언
  • js로 컴파일된 뒤에도 모듈 호출(import, require) 가능

namespace extend

  • 상위 namespace에서 하위 namespace를 위에서 참조하면 에러
  • 왜그럴까?

multi modules export

  • export하려는 함수가 interface를 사용하는 경우 interface도 export 필요
  • as 키워드를 이용해 export하는 모듈 이름 변경
