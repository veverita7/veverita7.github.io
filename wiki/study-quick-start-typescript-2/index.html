var 선언자 (76p)

  • 호이스팅(hoisting) 발생
  • 끌어올린다는 의미로 선언된 위치와 관계없이 스코프의 최상위로 끌어올려 같은 스코프라면 어디서든 호출
  • 함수 레벨 스코프를 지원해서 함수안에서 선언한 변수는 함수 안에서만 유효
  • 블록 레벨 스코프를 지원하지 않아서 블록 내부와 외부가 같은 스코프로 인식

let 선언자 (78p)

  • es6(es2015)에서 지원
  • 호이스팅 방지
  • 블록 레벨 스코프 지원
  • 같은 블록 내에서 같은 이름으로 중복 선언 불가

const 선언자 (80p)

  • let 선언자와 특성이 비슷
    • es6(es2015)에서 지원
    • 호이스팅 방지
    • 블록 레벨 스코프 지원
  • let 선언자와 차이는 상수라서 읽기 전용
  • 객체 리터럴 설정시 객체 재할당은 안되지만 속성으로는 변경 가능

타입 검사 (82p)

타입 검사 종류 언어
정적 타입 검사 (statically type checking) java, c
동적 타입 검사 (dynamically type checking) javascript
점진적 타입 검사 (gradually type checking) typescript, python

점진적 타입 검사는 컴파일 시간에 타입 검사를 수행하면서 필요에 따라 사입 선언의 생략을 허용한다.
